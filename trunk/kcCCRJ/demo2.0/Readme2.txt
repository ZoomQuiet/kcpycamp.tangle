kcCCRJ 

版本说明：
	kcCCRJ demo版本1.0  
	完成于2008年8月22日

产品概述：
	本产品kcCCRJ基于金山训练营的代码规范，是一个可用于检测代码是否符合代码规范要求的软件。
	kcCCRJ采用命令行输入，文本输出的方式，用户也可用它来进行批处理。每次可对单个文件或一个目录下所有c,cpp,h文件进行检测。
	另外，kcCCRJ提供了检测功能选项，用户可以根据需要选择其中几项来进行检测。
	本软件既可以给老师用于检测作业，也可以让学生进行自检。简直是居家旅行打家劫舍必备良药。

环境要求：
	要求使用的电脑已安装python

安装说明：
	下载本软件包并将其在任意目录下解压。为了方便使用，我们建议用户设置环境变量。
	环境变量设置方法：
		步骤:我的电脑->右键->属性->高级->环境变量->(系统变量)新建
		变量名:(随便)
		变量值:解压包中CCRJ.PY主程序所存放的路径
		例如
		变量名:CCRJ
		变量值:C:\Python25\CCRJ\（假如CCRJ.PY存放于C:\Python25\CCRJ\）
		那么在CMD下就可以直接用CCRJ.PY C:\TEST.CPP C:\（具体命令用途参照使用方法）


使用方法：
	kcCCRJ采取命令行方式运行，为了方便使用，我们建议用户设置环境变量。
	在控制台下，我们提供了以下各种命令。（以下是设置了环境变量后各命令写法，若没有更改环境变量，请把命令中第一个参数CCRJ.PY改成CCRJ.PY存放的完整路径。如：C:\Python25\CCRJ\CCRJ.PY）
	帮助命令
	CCRJ.PY --help
	
	检测单个文件：
	CCRJ.PY [文件名] [检测结果保存目录]
	如：CCRJ.PY C:\TEST.CPP C:\
	
	检测单个目录：
	CCRJ.PY [目录] [检测结果保存目录]
	如：CCRJ.PY C:\VC++ C:\
	
	若需要选择检测功能，可以在输入文件或目录检测命令时增加 -[选项集]
	如选择ACE检测功能：
	CCRJ.PY  C:\TEST.CPP  C:\  -ABC
	具体检测功能选项含义请参照检测功能选项的说明。
	
检测功能选项：
	本版本提供了以下检测功能选项：
	A：文件头注释检测
		用于检测文件是否包含文件头注释。重点检测是否包含文件名，创建人，创建时间，功能描述。
		这里的检测以金山训练营约定文件头注释为基准，具体参照如下：
		/* -------------------------------------------------------------------------
		//	文件名		：	KServiceControl.cpp
		//	创建者		：	方正圆
		//	创建时间	：	2008-8-17 0:25:29
		//	功能描述	：	
		//
		//	$Id: $
		// -----------------------------------------------------------------------*/
		这个功能点采用模糊匹配原则。
	B：关于{}问题
		具体检测内容：在{之前只允许有空格和Tab，后面可以有注释，但不可以有代码；与{对应的}必须在同一列
		
	C：合并行问题
		具体检测内容：循环，分支代码，判断条件与执行代码不可在同一行
		
	D：空格问题
		具体检测内容：if,while,for之后有且只有一个空格

	E：变量名命名规范
		具体检测内容：是否符合代码规范.pdf中附录一 的4变量名 中类型缩写的约定
		注意：
			对于缺省定义变量,将被认为不符合规范

		本版本尚未能解决这个检测的以下问题：
			1.不能检查类、结构、宏、枚举以及联合的类型定义
			2.不能检测多行定义的情况
			3.假设操作符号后面要有空格，否则检测出错（根据空格的相关代码规范。PS本版本尚未实现空格的这个规范）
			4.不能检测字符串和指针
			5..要求符合规范的变量最短的是类型加名字(前面字母是类型名,类型名过后的第一个字母必须大写)
			6.可以处理行(//)注释问题,但是不考虑块/**/注释问题
